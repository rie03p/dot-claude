# デバッグスキル

エラーやバグの原因を特定し、修正する。

## 引数の解析

`$ARGUMENTS` からエラーメッセージ、バグの説明、スタックトレースを読み取る。未指定ならユーザーに状況を聞く。

## 手順

### 1. 情報収集

エラーの種類に応じて情報を集める:

- **スタックトレースがある場合**: トレースに含まれるファイル・行番号を順に読む
- **再現手順がある場合**: 関連コードを特定して読む
- **曖昧な報告の場合**: Grep/Glob でキーワード検索し、関連コードを絞り込む

追加で確認すること:
- 最近の変更（`git log --oneline -20`, `git diff HEAD~5`）
- 関連する設定ファイル
- 依存パッケージのバージョン

### 2. 仮説の立案

収集した情報から原因の仮説を立てる:

```
## 調査結果

### 症状
（何が起きているか）

### 原因の仮説（可能性が高い順）
1. [仮説A] — 根拠: ...
2. [仮説B] — 根拠: ...

### 調査で判明した事実
- ...
```

### 3. 仮説の検証

最も可能性が高い仮説から検証する:

- コードを読んで裏付けを取る
- 可能なら再現を試みる（テスト実行、ログ確認）
- 仮説が外れたら次の仮説へ

### 4. 修正の実行

原因が特定できたら:

1. 修正方針をユーザーに説明
2. 承認を得てから修正を実施
3. 関連テストがあれば実行
4. テストがなければ、修正を検証するテストの追加を提案

### 5. 報告

- 根本原因の説明
- 修正内容
- 再発防止のための提案（あれば）
