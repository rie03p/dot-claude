# リファクタリングスキル

指定されたコードを分析し、リファクタリング案を提示して実行する。

## 引数の解析

`$ARGUMENTS` からファイルパスまたは関数名を読み取る。未指定ならユーザーに聞く。

## 手順

### 1. 対象コードの分析

対象ファイル/関数を読み込み、以下を分析する:

- コードの責務と依存関係
- 問題点の特定:
  - 重複コード
  - 長すぎる関数（目安: 50行以上）
  - 深いネスト（3階層以上）
  - 不明瞭な命名
  - 不要な複雑さ
  - 型安全性の問題
- 呼び出し元/呼び出し先の把握（影響範囲の特定）

### 2. リファクタリング案の提示

分析結果をもとに、具体的な改善案を優先度付きで提示する:

```
## リファクタリング案

### 高優先度
1. [具体的な変更内容] — 理由: ...
2. ...

### 中優先度
1. ...

### 影響範囲
- 変更されるファイル: ...
- テストへの影響: ...
```

ユーザーの承認を得てから次へ進む。

### 3. 段階的な実行

承認された項目を**1つずつ**実行する:

1. 変更を適用
2. 既存テストがあれば実行して壊れていないことを確認
3. 型チェック/リントがあれば実行
4. 次の変更へ

一度に全てを変更しない。各ステップで動作確認を挟む。

### 4. 完了報告

- 変更箇所のサマリー
- Before/After の比較（主要な変更のみ）
- テスト結果
- 追加で検討すべきリファクタリング（あれば）
