# PR レビュー表示スキル

指定された GitHub PR の差分を取得し、変更内容を構造化して表示する。コメントの生成や投稿は行わない。

## 引数の解析

`$ARGUMENTS` から PR 番号または URL を読み取る。未指定ならユーザーに聞く。

## 手順

### 1. PR 情報の取得

```bash
# PR のメタ情報
gh pr view <PR番号> --json title,body,state,author,baseRefName,headRefName,labels,reviewRequests,additions,deletions,changedFiles,comments,reviews

# 差分の取得
gh pr diff <PR番号>
```

### 2. 変更の整理

以下の構成で表示する:

**概要**
- タイトル / 作者 / ベースブランチ → ヘッドブランチ
- 状態 / ラベル / レビュアー
- ファイル数 / 追加行数 / 削除行数

**変更ファイル一覧**
- ファイルごとに変更種別（追加/変更/削除）と変更行数を表示
- 論理的なグループに分類する（例: フロントエンド/バックエンド/テスト/設定）。グループ化することで、レビュアーが変更の全体像を素早く把握できる。

**変更の要約**
- 何がどう変わったかを箇条書きで簡潔に説明する
- 影響範囲の概要を示す

**レビューコメント・会話**（存在する場合）
- 既存のレビューコメントを要約する

**気になるポイント**（あれば）
- 潜在的なリスクや確認すべき箇所を列挙する
- 判断はユーザーに委ねる（このスキルはあくまで情報提示が目的であり、approve/reject の判断は行わない）

### 3. 追加操作

ユーザーが求めた場合のみ:
- 特定ファイルの差分を詳細表示
- 関連する Issue やコミット履歴の表示

## 使用例

**例1**: ユーザーが「PR #42 の内容を確認して」と言った場合
→ `gh pr view 42` と `gh pr diff 42` を実行し、上記フォーマットで整理して表示する。

**例2**: ユーザーが「https://github.com/org/repo/pull/99 をレビューして」と言った場合
→ URLからPR番号を抽出し、同様に情報を取得・整理して表示する。

**例3**: ユーザーが「最新のPRどうなってる？」と言った場合
→ `gh pr list --limit 5` で最近のPRを一覧表示し、どのPRを詳しく見るか確認する。
