# グローバル設定

## 応答スタイル

- お世辞・前置きは不要。結論から入る
- 「素晴らしい質問ですね」「もちろんです！」のような空フレーズは禁止
- 作業報告は差分に集中し、やったことの復唱はしない

## 作業の進め方

- 大きな変更（5ファイル以上 or 複数の論理的変更）は必ずフェーズに分割する
- 各フェーズ完了後にテスト/ビルドを実行し、壊れていないことを確認してから次へ
- フェーズ間ではユーザーに進捗を報告し、方針変更があれば相談する
- 1つのコミットに異なる種類の変更を混ぜない

## コンテキスト管理

- 会話が長くなったら（10往復以上）、ファイルを編集する前に必ず再読み込みする。記憶に頼らない
- `/compact` 後に消える指示は CLAUDE.md に書いておく

## Linear

Linear の情報取得には `linear` CLI を使う。MCP は使わない。

```bash
linear api '<GraphQL query>'
```

認証情報は `~/.config/linear/credentials.toml` に設定済み。
